Windows collection foundation

The normalizer now understands common Windows Security / Sysmon / PowerShell shapes from JSON and XML. Use any collector that can forward Winlogbeat-style JSON or Event XML into ingest.

Windows
Supported IDs
4624, 4625, 4688, 4698, 4720, 4728/4732/4756, 5156/5157, 7045, 1102, Sysmon 1/3/13.
Transport
HTTP ingest with JSON payloads, or syslog/file forwarders that ship Windows Event XML in the message field.
Built-in detections
Logon failure burst, audit log cleared, privileged group membership change, suspicious PowerShell encoded command, service installation, user creation.
Collector script
deploy/windows/rdegon-siem-collector.ps1 posts Security/System/Application/Sysmon/PowerShell events directly into /ingest/json and can install a scheduled task.

Enrichment and SOC usage

CMDB and TI data are applied during writing, then reused in Event Console, investigations and batch detections.

CMDB
Maps observed hosts or IPs to owner, criticality, environment and service. These fields become searchable in events_view.
Threat intel
Matches IPs, hosts, users, processes and raw substrings. Matching events receive TI tags and searchable fields like ti_indicator and ti_provider.
Port monitoring
Firewall blocks and SSH events are normalized with target port context so analysts can track which ports are probed and what service is likely being targeted.
